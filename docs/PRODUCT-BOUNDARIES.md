# 제품 경계

## Cloud GPU Runner

회사 클라우드 크레딧의 잔액, 비용, 저장소, 서버와 GPU 작업을 운영하는 단일 제품입니다.

- 공개 UI: `public/`
- 서버 API: `api/`
- 공급자·비용·작업 구현: `lib/`
- 공용 CLI: `scripts/cloud-gpu.ps1`
- 운영 주소: https://cloud-gpu-runner.vercel.app
- 기준 저장소: https://github.com/yeohj0710/cloud-gpu-runner

## 멀티클라우드 실행기

- NAVER Object Storage를 공통 아티팩트 저장소로 사용합니다.
- RTX 5070 Ti 16GB로 합리적으로 처리 가능한 작업은 로컬에서 실행합니다.
- 유료 클라우드는 GPU 1장당 VRAM 48GB 이상만 사용합니다. NAVER L40S와 KakaoCloud A100은 허용하고 NAVER L4와 KakaoCloud T4는 차단합니다.
- `auto`는 NAVER의 300,000원 크레딧(2026-07-31 만료)을 먼저 활용하고, 이후 5,000,000원 크레딧(2027-04-30 만료)을 사용합니다. NAVER가 준비되지 않았으면 Kakao를 사용합니다.
- 모든 작업에 최대 실행시간, 예상·실제 비용, 완료 콜백, 취소 경로와 시간초과 정리가 존재합니다.
- 완료·실패·취소·시간초과 작업은 서버, 공인 IP와 임시 디스크를 반납합니다.
- 프로젝트는 사전 서명 URL만 받고 공급자 키를 받지 않습니다.

## 공개 화면과 제어면

- 공개 화면은 잔액, 합산 비용, 익명화된 작업 유형과 자원 반납 여부만 제공합니다.
- 원본 작업 ID, 명령, 오류, 저장 경로, 실행·취소 기능은 인증된 관리자 화면에서만 제공합니다.
- 공개 API와 제어 API의 경계를 유지하며 공개 화면을 실행 권한으로 사용하지 않습니다.

## Work Memory

화면 기록과 대화·음성을 검색하는 개별 GPU 활용 모듈입니다.

- 로컬 앱: `work_memory/`
- GPU worker: `gpu/`
- Cloud GPU Runner의 상위 제품명이 아니라 내부 활용 모듈입니다.

## 연구 이력

초기 크레딧 조사, 비용 근거와 클라우드 실험은 `docs/`, `experiments/`, `cloud-functions/`에 보존합니다. 별도 운영 저장소나 별도 대시보드는 만들지 않습니다.

## 레거시 식별자

`work-memory`가 포함된 세션 서명, 저장 버킷, SSH 키와 Python 패키지명은 기존 데이터·인증 호환을 위한 내부 값입니다. 사용자 화면의 제품명이나 별도 배포를 의미하지 않습니다.
